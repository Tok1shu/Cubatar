Cubatar provides a simple yet powerful API to fetch and render layered 3D avatars for Minecraft players. It handles various inputs including usernames, UUIDs, and Base64 encoded skin URLs, delivering high-quality renders with support for outer layers (hats, etc.).

## üåü Features

- **Layered Rendering**: Generates high-quality avatars that correctly render the second layer (such as hats and accessories) for a complete and authentic look.
- **Flexible Input**: seamlessly accepts:
    - Minecraft Usernames
    - UUIDs (auto-detected)
    - Base64 encoded skin URLs
- **Mojang Integration**: Directly fetches skins from Mojang's sessions API when needed.
- **High Performance**:
    - **Smart Caching**: Uses Caffeine caching to minimize upstream API calls and reduce latency.
    - **Asynchronous Processing**: Optimized for high throughput.
- **Modern Stack**: Built on the latest Java 21 and Spring Boot ecosystem.

## üöÄ API Documentation

### Get Player Avatar

`GET /v1/avatar/{input}`

Fetches the avatar for a given player.

**Parameters:**

- `input` (path variable): The Minecraft User Name, UUID, or Base64 encoded skin URL.
- `size` (query parameter, optional): The width/height of the generated image in pixels. Default is `64`.

**Example:**

```bash
curl "http://localhost:8080/v1/avatar/Notch?size=128"
```

## üì¶ Getting Started

### Prerequisites

- JDK 21 or higher installed.
- Docker (optional).

### Running Locally

1. **Clone the repository:**
   ```bash
   git clone https://github.com/Tok1shu/Cubatar.git
   cd Cubatar
   ```

2. **Build and Run:**
   ```bash
   ./gradlew bootRun
   ```

### Running with Docker

1. **Build the image:**
   ```bash
   docker build -t cubatar .
   ```

2. **Run the container:**
   ```bash
   docker run -p 8080:8080 cubatar
   ```

The application will start on port `8080`.
